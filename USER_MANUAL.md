# คู่มือการใช้งานระบบ SMART HR V1.0

## แนะนำระบบ (Introduction)
ระบบ SMART HR V1.0 เป็นระบบบริหารจัดการทรัพยากรบุคคลแบบครบวงจร (Complete HR Management System) ที่ออกแบบมาเพื่อรองรับการทำงานขององค์กรทั้งขนาดกลางและขนาดใหญ่ ประกอบด้วยฟังก์ชันการจัดการพนักงาน, ระบบลงเวลา, ระบบลางาน, เงินเดือน และผู้ช่วยอัจฉริยะ (T All BOT)

## ฟังก์ชันการทำงานหลัก (Core Features)
1. **หน้าโต๊ะทำงาน (Dashboard / Home)**: สรุปภาพรวมขององค์กร สถิติการมาทำงาน และรายการแจ้งเตือนสิทธิ์ที่รอการพิจารณา
2. **ระบบข้อมูลพนักงาน (Employee Directory)**: จัดเก็บประวัติ, ฐานเงินเดือน, และกระบวนการ Onboarding / Offboarding
3. **ระบบจัดการเวลาทำงาน (Time & Attendance)**:
   - บันทึกการเข้า-ออกงาน
   - ตรวจสอบกะการทำงาน (Shift Schedules)
   - คำนวณและตรวจสอบ OT (Overtime)
4. **ระบบจัดการการลา (Leave Management)**: 
   - ระบบอนุมัติวันหยุด
   - จัดการวันหยุดประจำปีของบริษัท
5. **ระบบค่าตอบแทนและเงินเดือน (Payroll)**: คำนวณเงินเดือน, เงินหัก, เงินบวก และออกสลิปเงินเดือนอัตโนมัติ
6. **ระบบการประเมิน (Appraisals)**: ประเมินพนักงานแบบ 360 องศา และตั้งเป้าหมาย KPI/OKR
7. **ผู้ช่วย AI (T All BOT Copilot)**: ใช้ความสามารถของ Gemini API ตอบคำถามด้านกฎหมายแรงงานและให้คำปรึกษา HR เบื้องต้นแบบ Real-time

## คำชี้แจงการจัดทางสถาปัตยกรรมและการปรับปรุง (Architecture & Upgrades)
ในเวอร์ชันนี้ มีการปรับเปลี่ยนและปรับระดับประสิทธิภาพ (Performance Scaling):
- โมดูลาร์โค้ด (Modularity): ทำการแยกคอมโพเนนต์ย่อยๆ ออกจากหน้าจอหลักที่มีขนาดใหญ่ เพื่อให้อ่านโค้ด จัดการ และซ่อมบำรุงได้ง่ายในระยะยาว ซึงรวมไปถึงการแยก Shared Components ต่างๆ ให้สามารถถูกนำไปใช้ใหม่ได้ทั้งหมด
- ฐานข้อมูล Firestore: เปลี่ยนการเชื่อมต่อระบบจัดการสิทธิ์และคลังข้อมูลต่างๆ จาก Mock/Local Storage ให้ไปใช้ Firebase Database จริง เพื่อรองรับ Concurrency และข้อมูลแบบ Real-time
- สคริปต์ความปลอดภัย (Firebase Rules): ปรับตั้งค่าสิทธิ์ให้เข้าถึงข้อมูลเฉพาะพนักงานที่ได้รับใบอนุญาตเท่านั้น

## การตั้งค่าแอปพลิเคชันเพื่อพัฒนา (Developer Setup)
1. ติดตั้ง Package Library: `npm install`
2. สร้างที่เก็บข้อมูล Configuration: `.env` โดยระบุค่าให้ครบถ้วนตาม `.env.example`
3. เชื่อมต่อ Firebase และกำหนด API Key ให้ Gemini
4. ใช้คำสั่ง เริ่มพัฒนา: `npm run dev` หรือ `npm run build` ในโหมดโปรดักชัน
